# Botium Toys Security Audit

## Project Overview

This project was completed as part of the Google Cybersecurity Professional Certificate.

The objective was to perform an internal security audit for Botium Toys, a fictional toy company experiencing business growth and increasing cybersecurity requirements.

Using the NIST Cybersecurity Framework (NIST CSF), I evaluated the organization's existing controls, identified security gaps, reviewed compliance requirements, and provided recommendations to improve the company's security posture.

---

## Audit Scope

The audit covered:

* Employee devices and workstations
* Internal network infrastructure
* Data storage and retention systems
* E-commerce systems
* Inventory management systems
* Legacy systems
* Physical security controls

---

## Risk Assessment Findings

Key risks identified:

* Excessive employee access to sensitive data
* Lack of encryption for customer payment information
* No Intrusion Detection System (IDS)
* No disaster recovery plan
* Weak password requirements
* Absence of centralized password management
* Inadequate asset classification
* Missing least privilege and separation-of-duty controls

Risk Score: 8/10 (High)

---

## Controls Assessment

### Existing Controls

* Firewall
* Antivirus software
* CCTV surveillance
* Physical locks
* Fire detection and prevention systems

### Recommended Controls

* Least Privilege
* Separation of Duties
* Encryption
* Intrusion Detection System (IDS)
* Disaster Recovery Planning
* Password Management System
* Formal Legacy System Management

---

## Compliance Review

The audit evaluated organizational alignment with:

* PCI DSS
* GDPR
* SOC 1
* SOC 2

Several compliance gaps were identified, primarily involving access control, encryption, and data protection practices.
